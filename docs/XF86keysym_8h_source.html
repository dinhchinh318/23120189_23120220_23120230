1 /*
2  * XFree86 vendor specific keysyms.
3  *
4  * The XFree86 keysym range is 0x10080001 - 0x1008FFFF.
5  *
6  * The XF86 set of keysyms is a catch-all set of defines for keysyms found
7  * on various multimedia keyboards. Originally specific to XFree86 they have
8  * been been adopted over time and are considered a "standard" part of X
9  * keysym definitions.
10  * XFree86 never properly commented these keysyms, so we have done our
11  * best to explain the semantic meaning of these keys.
12  *
13  * XFree86 has removed their mail archives of the period, that might have
14  * shed more light on some of these definitions. Until/unless we resurrect
15  * these archives, these are from memory and usage.
16  */

237 /*
238  * Reserved range for evdev symbols: 0x10081000-0x10081FFF
239  *
240  * Key syms within this range must match the Linux kernel
241  * input-event-codes.h file in the format:
242  * XF86XK_CamelCaseKernelName _EVDEVK(kernel value)
243  * For example, the kernel
244  * #define KEY_MACRO_RECORD_START 0x2b0
245  * effectively ends up as:
246  * #define XF86XK_MacroRecordStart 0x100812b0
247  *
248  * For historical reasons, some keysyms within the reserved range will be
249  * missing, most notably all "normal" keys that are mapped through default
250  * XKB layouts (e.g. KEY_Q).
251  *
252  * CamelCasing is done with a human control as last authority, e.g. see VOD
253  * instead of Vod for the Video on Demand key.
254  *
255  * The format for #defines is strict:
256  *
257  * #define XF86XK_FOO<tab...>_EVDEVK(0xABC)<tab><tab> |* kver KEY_FOO *|
258  *
259  * Where
260  * - alignment by tabs
261  * - the _EVDEVK macro must be used
262  * - the hex code must be in uppercase hex
263  * - the kernel version (kver) is in the form v5.10
264  * - kver and key name are within a slash-star comment (a pipe is used in
265  * this example for technical reasons)
266  * These #defines are parsed by scripts. Do not stray from the given format.
267  *
268  * Where the evdev keycode is mapped to a different symbol, please add a
269  * comment line starting with Use: but otherwise the same format, e.g.
270  * Use: XF86XK_RotationLockToggle _EVDEVK(0x231) v4.16 KEY_ROTATE_LOCK_TOGGLE
271  *
272  */
273 #define _EVDEVK(_v) (0x10081000 + _v)
