Detailed Description

Interface for stream protocol with manual buffer control.

Event methods, such as `create_server` and `create_connection`,
accept factories that return protocols that implement this interface.

The idea of BufferedProtocol is that it allows to manually allocate
and control the receive buffer.  Event loops can then use the buffer
provided by the protocol to avoid unnecessary data copies.  This
can result in noticeable performance improvement for protocols that
receive big amounts of data.  Sophisticated protocols can allocate
the buffer only once at creation time.

State machine of calls:

  start -> CM [-> GB [-> BU?]]* [-> ER?] -> CL -> end

* CM: connection_made()
* GB: get_buffer()
* BU: buffer_updated()
* ER: eof_received()
* CL: connection_lost()

def asyncio.protocols.BufferedProtocol.eof_received (   self) 
Called when the other end calls write_eof() or equivalent.

If this returns a false value (including None), the transport
will close itself.  If it returns a true value, closing the
transport is up to the protocol.

Reimplemented in asyncio.sslproto.SSLProtocol.

Called to allocate a new receive buffer.

*sizehint* is a recommended minimal size for the returned
buffer.  When set to -1, the buffer size can be arbitrary.

Must return an object that implements the
:ref:`buffer protocol <bufferobjects>`.
It is an error to return a zero-sized buffer.

Reimplemented in test.test_asyncio.test_buffered_proto.ReceiveStuffProto, and asyncio.sslproto.SSLProtocol.
